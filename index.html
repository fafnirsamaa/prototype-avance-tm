Insight clé

Le moment de bascule est l'étape « Considération » : c'est là que l'émotion s'effondre de +1 à 0, parce que la page produit ne fait que présenter de l'information — alors qu'Inès vient chercher de l'émotion. Le pivot du redesign doit donc se faire à cet endroit précis : transformer la page produit en moment d'expérience.

Livrable 2 — Parcours utilisateur cible

Carrés de Soie : la maison s'invite à l'écran

M2UX 2026
Parcours cible

Le parcours cible repositionne la page produit comme la pièce maîtresse de l'expérience, en remplaçant la grille catalogue par un récit scroll-couplé qui présente quatre carrés comme une mini-collection. Chaque transition mobilise une micro-interaction technique impossible à reproduire dans Figma classique.

01
Desktop / MobileArrivée

Arrivée sur la page

L'utilisatrice arrive depuis une newsletter ou un lien partagé. Le premier carré (« On est bien ici ! ») apparaît centré sur fond crème, pendant qu'un loader Hermès SVG décroît.

Micro-interaction — Loader Hermès
Trigger : Page load complete (textures Cloudinary réceptionnées)
Feedback : Barre de progression horizontale 0 → 140px en 0.9s ease power2.inOut, puis fade-out du loader 0.8s. Logo header en entrée subtile (translate Y −8px → 0, fade 0 → 1, 1.0s).

Premier moment Hermès assumé, pas un site e-commerce.

02
DesktopHover sur le carré

Le silk react sous le curseur

Inès passe la souris sur le carré. Un halo lumineux multi-couches suit le curseur, et la surface géométrique du tissu se déforme légèrement autour de la souris — comme une vraie soie qu'on toucherait.

Micro-interaction — Cursor sheen + silk displacement
Trigger : pointermove sur le canvas WebGL
Feedback : Vertex shader applique du noise simplex 2D localisé dans un radius UV de 0.45, eased par uMouseStrength (ramp 0 → 1 en 0.6s à l'entrée, 1 → 0 en 1.2s à la sortie). Fragment shader superpose un halo crème (rayon 0.42 UV, falloff pow(smoothstep, 1.4)) + un cœur gold-cream (rayon 0.16 UV, falloff pow(smoothstep, 1.6)).

Sensation tactile inédite, le carré semble vivant — moment signature impossible en classique Figma.

03
Desktop / MobileScroll vers carré 2

Tissage en temps réel du carré suivant

L'utilisatrice scroll vers le bas. Le carré dérive en arc à travers la page, billow en 3D comme une voile dans le vent, puis le carré suivant se tisse littéralement à sa place — un front horizontal sweep top-down avec des fils warp/weft visibles dans la zone de transition.

Micro-interaction — Weave shader + arc choreography
Trigger : Position de scroll (ScrollTrigger scrub: 2.4)
Feedback : Mesh position en lerp easeInOutQuart + arc windLift (rise fast, settle slow) ; rotation lagged en pow(t, 1.4) ; uMotionStrength drive un billow géométrique sin(uv.x·π)·sin(uv.y·π)·0.55 qui catche l'air comme une voile. Fragment shader sweep front = uMix * (1 + 2·band) − band avec fils warp 340 cycles + weft 320 cycles dans une enveloppe smoothstep(band·1.5, 0, |dist|), plus shimmer leading-edge 0.018 UV.

Découverte progressive narrative, jamais de bouton « next » plat.

04
Desktop / MobileScroll-stop entre 2 carrés

Magnet vers le carré le plus proche

Si Inès s'arrête entre deux carrés, l'expérience l'aimante doucement vers le plus proche — sans jamais combattre un scroll actif.

Micro-interaction — ScrollTrigger snap delayed
Trigger : Scroll inactif depuis 180 ms (delay GSAP)
Feedback : GSAP tween de window.scrollY vers le snap point le plus proche parmi [0, 1/3, 2/3, 1] de la séquence d'actes ; durée min 0.4s max 0.9s, ease power2.inOut, inertia: false.
05
Desktop / MobileDissolve vers la grille

Le dernier carré se défait dans la page

Lorsque la grille collection approche, le carré actif (Paris qui Roule) se détisse littéralement par le bas — symétrique au tissage, vocabulaire visuel cohérent.

Micro-interaction — Unweave shader
Trigger : ScrollTrigger sur #collection approchant top center
Feedback : uTexB remplacé par une creamTexture (1×1 #FFFCF7), uWeaveDirection = −1 bascule le sens du sweep en bottom-up, uMix progresse 0 → 1 sur le scroll. Les fils warp/weft restent visibles à l'avant pendant que le carré disparaît dans la couleur de la page.

Symétrie tisser/détisser — métaphore complète, le carré naît et retourne au métier.

06
Desktop / MobileAjout au panier depuis grille

Conversion silencieuse

La grille présente les 4 carrés avec sélecteur de quantité et bouton « Ajouter ». Au clic, confirmation visuelle non-intrusive.

Micro-interaction — Add confirm + cart icon pulse
Trigger : click sur bouton « Ajouter »
Feedback : Bouton swap texte → « Ajouté ✓ » en cream-light pendant 1.6s. Simultanément, GSAP yoyo sur l'icône cart header : scale: 1 → 1.15 → 1 en 0.18s × 2. Compteur cart-count incrémenté avec aria-live="polite".
07
Desktop / MobilePage produit dédiée

Click sur une carte → page produit

Pour examen détaillé, click sur la carte ouvre une page produit dédiée via routing hash-based, avec photo plein cadre, designer, dimensions, composition, et accès rapide aux autres carrés de la collection.

Micro-interaction — Hash routing + view swap
Trigger : click sur .product-card → window.location.hash = "#product/c01"
Feedback : hashchange handler hide .main-element, show .route-view#product-view populée avec les data du carré. WebGL canvas opacity 0, sticky-cta hidden. Body class has-route-view pour cascade des règles.

Inès peut prendre son temps sur un carré sans perdre le contexte de collection.

08
Desktop / MobilePersistance

Le panier survit à un refresh

Inès ferme l'onglet pour répondre à un email. Quand elle revient, le carré est encore dans son panier — le compteur header est déjà à jour, le drawer pre-rempli.

Micro-interaction — localStorage hydration
Trigger : Page load
Feedback : readCart() parse hermes_carre_cart_v1 → schéma { items: [{ id, name, price, qty }] }. refreshCartUI() met le compteur à jour et pre-popule le drawer avec les thumbnails Cloudinary correspondantes.

Pas de panique de perte, reprend l'achat où elle s'est arrêtée.
